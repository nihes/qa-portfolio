# Test Cases — Account Registration, Login, Logout & Password Reset

Reference SUTs: [saucedemo.com](https://www.saucedemo.com/) (login/locked-user
reference), [automationexercise.com](https://automationexercise.com/)
(registration/password-reset reference). Substitute the actual project SUT
and its real test data/URLs when reusing this suite.

| ID | Title | Preconditions | Steps | Test Data | Expected Result | Priority | Type |
|---|---|---|---|---|---|---|---|
| TC-AUTH-001 | New user can register successfully with valid data | User is not logged in; the email used is not already registered | 1. Navigate to the Signup/Register page. 2. Fill in all required fields with valid, unique data. 3. Accept any required terms/checkboxes. 4. Submit the form. | Unique email e.g. `qa.portfolio+<timestamp>@example.com`; valid name, password meeting policy, and any other required fields | Account is created successfully; user is either automatically logged in or shown a clear success/confirmation message; the new account can subsequently log in with the same credentials | High | Functional |
| TC-AUTH-002 | Registration is rejected when using an already-registered email | An account already exists for a given email | 1. Navigate to the Signup/Register page. 2. Enter the email of an existing account. 3. Fill remaining fields validly. 4. Submit. | Email already registered, e.g. an existing test account's email | A clear error message is shown (e.g. "Email already exists"); no duplicate account is created; user remains on the registration form with entered data preserved where reasonable | Med | Negative |
| TC-AUTH-003 | Login succeeds with valid credentials | A valid, active user account exists | 1. Navigate to the Login page. 2. Enter the valid username/email and password. 3. Submit. | SauceDemo `standard_user` / valid password, or an equivalent valid account | User is logged in and redirected to the expected post-login page (e.g. products/dashboard); username or account indicator is visible in the header; session is established (protected pages become accessible) | High | Functional |
| TC-AUTH-004 | Login is rejected with an incorrect password | A valid, active user account exists | 1. Navigate to the Login page. 2. Enter a valid username/email with an incorrect password. 3. Submit. | Valid username + password = `WrongPassword123` | A generic error message is shown (e.g. "Username and password do not match") without revealing whether the username or password specifically was wrong; user is not logged in; no session/cookie is established | High | Negative |
| TC-AUTH-005 | Login is rejected for a locked-out user with an explanatory message | A user account exists in a locked/disabled state | 1. Navigate to the Login page. 2. Enter the locked account's valid username and correct password. 3. Submit. | SauceDemo `locked_out_user` / `secret_sauce` | A specific error message is shown indicating the account is locked/blocked (e.g. "Sorry, this user has been locked out"); user is not logged in | High | Negative |
| TC-AUTH-006 | Login is rejected for a non-existent username | No account exists for the given username | 1. Navigate to the Login page. 2. Enter a username that has never been registered, with any password. 3. Submit. | Username = `nonexistent_user_9999` | A generic "invalid credentials" style error is shown (not "user not found", to avoid leaking account existence); user is not logged in | Med | Negative |
| TC-AUTH-007 | Logout ends the session and blocks access to protected pages | User is currently logged in with an active session | 1. Click the account menu / "Logout" control. 2. Confirm logout if prompted. 3. Attempt to navigate directly (via URL) to a page that requires authentication (e.g. account dashboard, order history). | Logged-in user session | User is redirected to the login page (or public home page); the previously accessible protected page now redirects to login instead of rendering; session cookie/token is invalidated (a manual back-navigation to a protected page does not restore access) | High | Functional |
| TC-AUTH-008 | Password reset request for a valid registered email sends reset instructions | An account with a known, valid email exists; user is not logged in | 1. Navigate to "Forgot Password" from the Login page. 2. Enter the registered email address. 3. Submit. | A valid registered test account's email | A confirmation message is shown (e.g. "If this email is registered, reset instructions have been sent") without confirming/denying account existence explicitly; a password reset email/link is sent to that inbox in environments where email delivery is testable (e.g. via Mailhog) | Med | Functional |
