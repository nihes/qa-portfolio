# Test Cases — Checkout

Reference SUTs: [saucedemo.com](https://www.saucedemo.com/),
[automationexercise.com](https://automationexercise.com/). Substitute the
actual project SUT and its real test data/URLs when reusing this suite.

| ID | Title | Preconditions | Steps | Test Data | Expected Result | Priority | Type |
|---|---|---|---|---|---|---|---|
| TC-CHK-001 | Successful checkout with valid data (happy path) | User has at least one in-stock item in the cart; on the cart/checkout entry page | 1. Open cart. 2. Click "Proceed to Checkout". 3. Enter a complete, valid shipping address. 4. Select a shipping method. 5. Enter valid payment details. 6. Review order summary. 7. Click "Place Order" / "Confirm Payment". | 1× in-stock item, qty 1; valid full address (e.g. Automation Exercise test address); sandbox test card marked "success" | Order is placed successfully; an order confirmation page is shown with an order number; a confirmation email is triggered (if applicable); order total matches cart total + shipping + tax − discount | High | Functional |
| TC-CHK-002 | Checkout is blocked when cart is empty | Cart is empty (no items added, or all items removed) | 1. Navigate directly to the checkout URL/page with an empty cart. | N/A | User is redirected to the cart/home page, or shown a clear "Your cart is empty" message; checkout form is not rendered; no order is created | High | Negative |
| TC-CHK-003 | Checkout blocked when required address field is missing | User has an item in cart and has reached the address step of checkout | 1. Leave a required field empty (e.g. "Postal Code" or "Address Line 1"). 2. Fill all other required fields with valid data. 3. Attempt to proceed to the next step. | Address with Postal Code field blank; all other fields valid | Inline field-level validation error is shown next to the missing field; user is not allowed to proceed to payment; no order is created | High | Negative |
| TC-CHK-004 | Checkout rejects invalid field format in address | User has an item in cart and has reached the address step of checkout | 1. Enter an invalid value in a format-validated field (e.g. letters in a numeric ZIP/postal code field, or a malformed phone number). 2. Fill all other fields validly. 3. Attempt to proceed. | Postal code = `ABC$$`; phone = `123` | Validation error is shown for the invalid field describing the expected format; user cannot proceed until corrected | Med | Negative |
| TC-CHK-005 | Valid coupon code is applied and reduces the total correctly | User has item(s) in cart with a known subtotal; a currently active coupon code exists for the store | 1. Go to cart or checkout coupon field. 2. Enter the valid coupon code. 3. Click "Apply". | Cart subtotal = 40.00; coupon = 10% off, code e.g. `SAVE10` | Coupon is accepted with a success message; discount line item appears in the order summary; new total = subtotal − (10% of subtotal) [+ tax/shipping as applicable]; total recalculates immediately without a full page reload requirement | High | Functional |
| TC-CHK-006 | Invalid/expired coupon code is rejected | User has item(s) in cart; a coupon code known to be invalid or expired is available for testing | 1. Enter the invalid/expired coupon code in the coupon field. 2. Click "Apply". | Coupon code = `INVALID10` (non-existent) or a coupon past its `expiration_date` | An error message is shown (e.g. "Coupon code is invalid or has expired"); no discount is applied; order total remains unchanged | High | Negative |
| TC-CHK-007 | Order total is calculated correctly across subtotal, discount, shipping, and tax | User has 2+ distinct items in cart, each with a known unit price and quantity; a valid coupon is applied; a shipping method with a known cost is selected | 1. Add items with known prices/quantities to cart. 2. Apply a valid coupon. 3. Select a shipping method with a non-zero cost. 4. Proceed to the order review/summary step. | Item A: 2× 15.00; Item B: 1× 9.99; coupon: 10% off subtotal; shipping: 5.00 flat | Subtotal = 39.99; discount = 4.00 (rounded per store's rounding rule); shipping = 5.00; grand total = subtotal − discount + shipping [+ tax if applicable], and every line item matches this arithmetic exactly (no floating-point drift, no missing line) | High | Boundary |
| TC-CHK-008 | Guest checkout completes without requiring account creation | User is not logged in; cart has at least one item | 1. Add item to cart while logged out. 2. Proceed to checkout. 3. Choose "Checkout as Guest" (or equivalent) when prompted. 4. Complete address, shipping, and payment steps with valid data. 5. Place order. | Valid guest email + shipping address; sandbox test card marked "success" | Checkout completes successfully without forcing account registration; order confirmation is shown; confirmation is sent to the guest email if the store supports email confirmation | Med | Functional |
| TC-CHK-009 | Payment failure is handled gracefully and cart/order state is preserved | User is logged in or checking out as guest with a valid cart; has reached the payment step | 1. Enter payment details known to trigger a decline in the payment sandbox. 2. Click "Place Order" / "Confirm Payment". | Sandbox test card marked "decline"/"insufficient funds" | A clear, user-facing error message is shown (no raw gateway/technical error leaked to the UI); no order is created; cart contents and entered address are preserved so the user can retry without re-entering everything | High | Negative |
| TC-CHK-010 | Order confirmation page displays accurate order details after successful checkout | Checkout has just been completed successfully (follows on from TC-CHK-001 or TC-CHK-008) | 1. Complete a successful checkout. 2. On the confirmation page, review the displayed order number, items, quantities, prices, shipping method, and grand total. | Order placed in TC-CHK-001/TC-CHK-008 | Confirmation page shows a unique order number; all items/quantities/prices match what was in the cart at checkout; grand total matches the value calculated at the review step; page persists/is retrievable on refresh (does not error or duplicate the order) | Med | Functional |
